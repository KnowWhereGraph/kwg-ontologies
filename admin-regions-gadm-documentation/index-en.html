Overview of the raw datasets back to ToC

  • Name of dataset: Global Adminstrative Areas (GADM) Dataset Series & Federal Information Processing Series (FIPS) Dataset
  • Source Agency: GADM & U.S. Census Bureau
  • Data source location: GADM maps and data & ANSI and FIPS Codes
  • Metadata description: GADM metadata & FIPS metadata
  • Format of data returned: Geopackage and Geodatabase for the world subdivisions at the finest level; Geopackage, Shape, GeoJSON, and KMZ for each country at every level of subdivision; text files for databases about FIPS codes that are separated by the character |.
  • Data update interval:L atest release is from July 2022 for GADM, latest release is from 2020 for FIPS codes
  • General comments: It is worth mentioning that some counties containing the string "Lake", such as " Lake Michigan", are missing an associated FIPS code in these data.

Administrative Regionc back to ToC or Class ToC

IRI: http://stko-kwg.geog.ucsb.edu/lod/ontology/AdministrativeRegion

This class denotes administrative areas of all countries, at all levels of sub-division from the Global Administrative Areas (GADM) dataset series. The dataset series is comprised of the six administrative levels global datasets for: National (level 0), State/province/equivalent (level 1), County/district/equivalent (level 2), Commune/municipality/equivalent (level 3) and smaller Levels 4 and 5.

Administrative Region (level 0)c back to ToC or Class ToC

IRI: http://stko-kwg.geog.ucsb.edu/lod/ontology/AdministrativeRegion_0

The Country boundaries at level 0 dataset is part of the Global Administrative Areas (GADM) 3.6 vector dataset series which includes distinct datasets representing administrative boundaries for all countries in the world. GADM makes use of high spatial resolution images and an extensive set of attributes to map administrative areas at all levels of political sub-division. The National level 0 attributes comprise country name in English and ISO 3166-1 alpha3 coding. Please read the GADM - Global Administrative Areas dataset series metadata (https://gadm.org/metadata.html) for more information.

has FIPSdp back to ToC or Data Property ToC

IRI: http://stko-kwg.geog.ucsb.edu/lod/ontology/hasFIPS

This property is used to denote the FIPS code of an administrative region within the US. Federal Information Processing System (FIPS) Codes are numbers which uniquely identify geographic areas. The number of digits in FIPS codes vary depending on the level of geography. State-level FIPS codes have two digits, county-level FIPS codes have five digits of which the first two are the FIPS code of the state to which the county belongs. When using the list below to look up county FIPS codes, it is advisable to first look up the FIPS code for the state to which the county belongs. This will help you identify the right section of the list while scrolling down, which can be important since there are over 3000 counties and county-equivalents (e.g.independent cities, parishes, boroughs) in the United States.

has GIDdp back to ToC or Data Property ToC

IRI: http://stko-kwg.geog.ucsb.edu/lod/ontology/hasGID

This property is used to denote the GID of an administrative region. The GID starts with the three letter ISO 3166-1 alpha-3 country code. ISO 3166-1 alpha-3 codes are three-letter country codes defined in ISO 3166-1, part of the ISO 3166 standard published by the International Organization for Standardization (ISO), to represent countries, dependent territories, and special areas of geographical interest.
